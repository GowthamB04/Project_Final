================================================================================
WORKFLOW IMPACT:
================================================================================

Claim Submission Flow (Now Fully Automated):

1. User submits claim request with treatment details including:
   - Doctor info (name, specialization, qualification, experience)
   - Hospital info (name, address, phone)
   - Treatment details (diagnosis, description, amount, date)
   - Documents (uploaded files)

2. Backend ClaimServiceImpl.saveClaim() processes:
   ✓ Creates Hospital entity (if new hospital provided)
   ✓ Creates Doctor entity (if new doctor provided, links to hospital)
   ✓ Saves Treatment with FK relationships to Doctor & Hospital
   ✓ Saves Claim with link to Treatment
   ✓ Calculates recommendation score
   ✓ Validates policy and user

3. Data stored in normalized form:
   - treatments table: treatment_id, diagnosis, amount, date, user_id, doctor_id, hospital_id
   - doctors table: doctor_id, name, specialization, qualification, experience, hospital_id
   - hospitals table: hospital_id, name, type, address, phone
   - No redundant copies of doctor/hospital data

4. Approver can cross-verify against documents and DB data
5. Payment processing uses data from primary entities
