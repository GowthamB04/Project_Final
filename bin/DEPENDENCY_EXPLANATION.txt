1. SPRING BOOT STARTER DEPENDENCIES
================================================================================

1.1 spring-boot-starter-data-jpa
------------------------------------
GROUP ID: org.springframework.boot
ARTIFACT ID: spring-boot-starter-data-jpa
PURPOSE: 
  - Provides Spring Data JPA for database access and ORM (Object-Relational Mapping)
  - Includes Hibernate ORM framework automatically
  - Simplifies database operations with Repository interfaces
  
USAGE IN PROJECT:
  - Manages entity relationships and database interactions
  - JpaRepository interface is extended by all repository classes
  - Handles CRUD operations (Create, Read, Update, Delete)
  - Entity classes use @Entity, @OneToMany, @ManyToOne annotations
  - Executes SQL queries automatically

EXAMPLE USAGE:
  @Repository
  public interface UserRepository extends JpaRepository<User, Long> {
      Optional<User> findByUsername(String username);
  }

WHY NEEDED:
  - Eliminates manual SQL query writing
  - Provides automatic database transaction management
  - Supports Hibernate for ORM (Object-Relational Mapping)
  - Improves code maintainability and reduces boilerplate


1.2 spring-boot-starter-validation
----------------------------------
GROUP ID: org.springframework.boot
ARTIFACT ID: spring-boot-starter-validation
PURPOSE:
  - Provides input validation annotations and validation framework
  - Validates HTTP request bodies before processing
  
USAGE IN PROJECT:
  - Can be used for validating entity fields
  - Ensures data consistency before database operations
  - Validates request parameters

EXAMPLE USAGE:
  @NotEmpty(message = "Email cannot be empty")
  private String email;
  
  @Min(value = 18, message = "Age must be at least 18")
  private Integer age;

WHY NEEDED:
  - Prevents invalid data from entering the database
  - Provides automatic validation messages
  - Reduces manual validation code


1.3 spring-boot-starter-webmvc
------------------------------
GROUP ID: org.springframework.boot
ARTIFACT ID: spring-boot-starter-webmvc
PURPOSE:
  - Provides Spring MVC (Model-View-Controller) framework
  - Enables REST API development
  - Automatically configures Tomcat server
  
USAGE IN PROJECT:
  - Creates REST API controllers using @RestController
  - Maps HTTP requests to controller methods
  - Handles JSON serialization/deserialization
  - Enables @RequestMapping, @GetMapping, @PostMapping, etc.

EXAMPLE USAGE:
  @RestController
  @RequestMapping("/api/users")
  public class UserController {
      @PostMapping("/login")
      public ResponseEntity<Map<String, Object>> login(@RequestBody Map<String, String> loginRequest) {
          ...
      }
  }

WHY NEEDED:
  - Essential for REST API development
  - Provides automatic JSON conversion (Jackson included)
  - Enables HttpStatus and ResponseEntity for HTTP responses
  - Handles cross-origin requests (@CrossOrigin)


1.4 spring-boot-devtools
------------------------
GROUP ID: org.springframework.boot
ARTIFACT ID: spring-boot-devtools
SCOPE: runtime (only during development)
PURPOSE:
  - Enables automatic restart of application during development
  - Provides live reload for static resources
  - Improves developer productivity
  
USAGE IN PROJECT:
  - Automatically restarts server when code changes
  - No need to manually stop and start application
  - Speeds up development cycle

WHY NEEDED:
  - Reduces development time
  - Provides instant feedback on code changes
  - Only runs during development (not in production)

2.1 mysql-connector-j
--------------------
GROUP ID: com.mysql
ARTIFACT ID: mysql-connector-j
SCOPE: runtime
PURPOSE:

CONFIGURATION:

  spring.datasource.username=root
  spring.datasource.password=root

WHY NEEDED:
  - MySQL database is chosen for this project
  - JDBC driver is required for database connectivity
  - Handles SQL command execution
  - Manages database connection pooling

3. SECURITY DEPENDENCIES
================================================================================

3.1 spring-security-crypto
--------------------------
GROUP ID: org.springframework.security
ARTIFACT ID: spring-security-crypto
PURPOSE:
  - Provides password hashing and encoding functionality
  - Implements BCrypt password encoding algorithm
  - Used for secure password storage without JWT/full Spring Security
  
USAGE IN PROJECT:
  - Password hashing during user creation
  - Password verification during login
  - BCryptPasswordEncoder bean automatically available

EXAMPLE USAGE:
  private final BCryptPasswordEncoder passwordEncoder = new BCryptPasswordEncoder();
  
  // Hashing password before saving
  user.setPassword(passwordEncoder.encode(user.getPassword()));
  
  // Verifying password during login
  if (passwordEncoder.matches(password, hashedPassword)) {
      // Password is correct
  }

WHY NEEDED:
  - Security requirement: passwords must never be stored in plain text
  - BCrypt is industry-standard password hashing algorithm
  - Provides salt-based hashing for protection against rainbow table attacks
  - Requirement specified: "Password must be hashed using BCryptPasswordEncoder"

6. JAKARTA PERSISTENCE (JPA Replacement for javax.persistence)
================================================================================

TRANSITION NOTES:
- Project uses jakarta.persistence.* (not javax.persistence.*)
- This is required for Spring Boot 4.0 and Java 17
- Spring Data JPA automatically includes this

IMPORTS USED:

NOT USED (deprecated in Spring Boot 4):
  import javax.persistence.*;

WHY JAKARTA:
  - Spring Boot 4+ requires Jakarta Enterprise Edition
  - Better namespace organization
  - Aligns with modern Java standards

8. SECURITY CONSIDERATIONS
================================================================================

1. NO JWT INCLUDED:
   - Project uses simple username/password authentication
   - Status quo: Users login with credentials stored in database
   - Password hashed with BCrypt
   - No token-based authentication (as per requirements)

2. NO FULL SPRING SECURITY:
   - Only spring-security-crypto included
   - Full Spring Security framework not included
   - Reduces complexity (as per requirements)

3. BCRYPT CONFIGURATION:
   - Default strength: 10 rounds
   - Higher = slower but more secure
   - Configurable but defaults are good

EXAMPLE STRENGTH:
  BCryptPasswordEncoder passwordEncoder = new BCryptPasswordEncoder(10);
  // 10 is default, can be changed to 12 or higher for more security

9. CORS (Cross-Origin Resource Sharing)
================================================================================

INCLUDED IN: spring-boot-starter-webmvc
CONFIGURATION: @CrossOrigin(origins = "*") on all controllers

This allows:
  - Angular frontend (or any frontend) to make requests
  - Frontend on different port/domain can access API
  - CORS headers automatically added to responses

PRODUCTION NOTE:
  For production, change origins = "*" to specific allowed origins:
  @CrossOrigin(origins = {"http://localhost:4200", "http://yourdomain.com"})


================================================================================
10. HIBERNATE CONFIGURATION
================================================================================

Properties used:
  spring.jpa.database-platform=org.hibernate.dialect.MySQL8Dialect
    - Tells Hibernate to use MySQL 8 dialect for SQL generation
  
  spring.jpa.hibernate.ddl-auto=update
    - Automatically creates/updates tables based on entities
    - Options: create, update, create-drop, validate, none
  
  spring.jpa.show-sql=true
    - Logs SQL statements to console (for debugging)
    - Disable in production for performance

DDL-AUTO BEHAVIOR:
  "update" = Creates tables if not exist, adds new columns to existing tables
           = Does NOT drop tables or columns

12. PRODUCTION VS DEVELOPMENT
================================================================================

DEV ENVIRONMENT:
  - spring-boot-devtools enabled (auto-restart)
  - spring.jpa.show-sql=true (logs SQL)
  - logging.level...=DEBUG (verbose logging)
  - CORS: origins = "*" (all origins allowed)

PRODUCTION ENVIRONMENT:
  - spring-boot-devtools disabled
  - spring.jpa.show-sql=false (performance)
  - logging.level...=INFO (minimal logging)
  - CORS: restricted to specific origins
  - Database credentials from environment variables
  - SSL enabled for HTTPS

15. TROUBLESHOOTING DEPENDENCY ISSUES
================================================================================

Issue: MySQL driver not found
Solution: Verify mysql-connector-j in pom.xml and run "mvn clean install"

Issue: Entity annotations not recognized
Solution: Ensure jakarta.persistence imports are used, not javax.persistence

Issue: BCryptPasswordEncoder not found
Solution: Add spring-security-crypto dependency

Issue: @RestController not working
Solution: Ensure spring-boot-starter-webmvc is included

Issue: Database connection timeout
Solution: Verify MySQL is running and credentials in application.properties
